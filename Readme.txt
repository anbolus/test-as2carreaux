Consignes ---------------------------------------------------------------------------------------------------

Corrigez et améliorez cette micro-application de messagerie.
Basée sur les langages html, css, javascript, ajax, jquery et php.

Vous le découvrirez dans le code, cette application simpliste de messagerie embarque une interface du
siècle dernier avec de potentielles erreurs et failles de sécurité.

--------------------------------------------------------------------------------------------------------------

Pour cette mission, vous devrez effectuer les tâches suivantes :

- Sécurisation du code contre d'éventuelles attaques.

- L'ajout d'une interface front respectant avec un UX / UI agréable

- Ajout d'un système de connexion basique (combo. user / password) qui remplacera l'input utilisateur "#name-input"

- Possibilité de retirer les messages précédemment postés.

- Possibilité de modifier un message précédemment posté.

- (OPTIONNEL) Ajout d'un websocket permettant le broardcast des messages (évite le lancement de la fonction "retrieveMessages" toutes les 2 secondes)


De façon générale, ajoutez toutes les idées vous passant par la tête pour compléter cette mini-app de messagerie. (ajout de photo de profil, profil utilisateur, grade d'utilisateur, ajout d'images dans le chat, etc.)
Le but étant d'exploiter L'INTÉGRALITÉ de vos compétences, alors n'hésitez pas à faire évoluer le code dans la direction qui, selon vous, sera la meilleure !


Une attention doit être portée sur...

1. Sécurité : il est essentiel de sécuriser l'application contre les vulnérabilités de sécurité telles que les injections SQL.
Utilisez des requêtes préparées et validez / nettoyez les entrées utilisateur pour éviter les attaques d'injection SQL.


2. Code propre : utilisez des pratiques de codage propres et des conventions de codage pour maintenir un code lisible et facile à comprendre.


3. Optimisation : optimisez le code pour améliorer les performances et la vitesse de chargement de l'application.


4. Accessibilité : assurez-vous que l'application est accessible pour les utilisateurs avec des besoins spécifiques en utilisant les bonnes techniques d'accessibilité pour le développement web.


5. Le bon sens : attention à ne pas recréer un facebook-like, limitez vous à l'exposition des compétences que vous avez acquis lors de votre formation / auto-formation.
